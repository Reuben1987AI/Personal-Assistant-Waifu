## Lessons Learned (IMPORTANT)
1. **`transparent: true` in tauri.conf.json causes click-through on Linux/WebKitGTK** тАФ the entire window becomes non-interactive. Set to `false` for dev.
2. **`withGlobalTauri: true` is required** in `tauri.conf.json` `app` section to inject `window.__TAURI__` into the webview. Without it, `main.js` crashes silently on line 1.
3. **Tauri v2 custom commands do NOT need explicit permissions in capabilities** тАФ app-level commands are automatically available with just `core:default`.
4. **`tokio-tungstenite` needs `native-tls` feature** for WSS connections тАФ add `features = ["native-tls"]` in Cargo.toml.
5. **Qwen-Omni-Realtime is NOT available in US Virginia** тАФ only Singapore and China (Beijing). Use `wss://dashscope-intl.aliyuncs.com/api-ws/v1/realtime` (international/Singapore endpoint).
6. **Each region requires its own API key** тАФ US Virginia key won't work for Singapore. User switched to Singapore key.
7. **Audio format must be specific**: `"pcm_16000hz_mono_16bit"` for input, `"pcm_24000hz_mono_16bit"` for output (not just `"pcm"`).
8. **Turn detection type is `"server_vad"`** not `"semantic_vad"` per the official SDK.
9. **`cpal::Stream` is not `Send`** тАФ can't be stored in async structs or moved across threads. Use `std::mem::forget(stream)` to keep it alive.
10. **Docker + inotify = broken** тАФ `cargo watch` polling works but `bunx tauri dev`'s built-in watcher doesn't detect host filesystem changes in Docker volumes.
11. **Cargo registry must be persisted via Docker volumes** тАФ without `-v waifu-cargo-registry:/usr/local/cargo/registry`, every container restart re-downloads all crates.
12. **X11 auth requires `xhost +local:docker`** + mounting XAUTHORITY file.
13. **JSON doesn't support `//` comments** тАФ don't add them to `tauri.conf.json`.
