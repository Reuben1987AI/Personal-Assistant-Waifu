## Wake detection (done)
Energy-gated + leading-silence framing in `src-tauri/src/main.rs::run_voice_agent`. Per-chunk RMS detects speech start/end (`RMS_SPEECH_THRESHOLD = 250`, 800ms trailing-silence confirmation). On end-of-speech, predicts once on `[800ms zero] + [last 1.2s of speech]` padded to 2s (the lead silence is critical â€” model was trained on TTS positives with ~800ms lead; without it the same audio scores 0.008, with it 0.66). `WAKE_THRESHOLD = 0.45`. Verified via `src-tauri/examples/test_wakeword.rs` against `src-tauri/kas1.pcm` (`cargo run --example test_wakeword -- kas1.pcm {slide|live|isolated}`).

Caveat: model is undertrained (small, 50k steps, 10k TTS samples). Currently reliable but depends on the lead-silence framing crutch. If misses become frequent, retrain with `medium` / 150k steps / 30k samples â€” see `wakeword-configs/kassandra.yaml`.

## Lessons Learned (still useful â€” don't retry these)
1. `transparent: true` in tauri.conf.json causes click-through on Linux/WebKitGTK â€” set `false`.
2. `withGlobalTauri: true` required in tauri.conf.json `app` section to inject `window.__TAURI__`.
3. Tauri v2 custom commands do NOT need explicit permissions â€” `core:default` is enough.
4. `tokio-tungstenite` needs `native-tls` feature for WSS.
5. Qwen-Omni-Realtime is NOT in US Virginia â€” intl/sg/cn only. intl: `wss://dashscope-intl.aliyuncs.com/api-ws/v1/realtime`. Each region needs its own API key.
6. Audio format: `pcm_16000hz_mono_16bit` in, `pcm_24000hz_mono_16bit` out.
7. `turn_detection.type = semantic_vad` for qwen3.5-omni models.
8. `cpal::Stream` is not `Send` â€” use `std::mem::forget(stream)` to keep mic + speaker alive for process lifetime.
9. Docker + inotify = broken â€” `bunx tauri dev` doesn't detect host FS changes in volumes. Run `cargo build` in the container manually before `make dev-run`, or the old binary runs.
10. Cargo registry must be persisted via Docker volumes (`waifu-cargo-registry`, `waifu-cargo-git`).
11. X11 auth requires `xhost +local:docker` + mounting XAUTHORITY.
12. JSON doesn't support `//` comments â€” don't add to tauri.conf.json.
13. WebKitGTK Web Audio API is broken in this Docker setup â€” play audio in Rust via cpal `speaker.rs`, not in the frontend.
14. `response.audio.delta` carries audio in the `delta` field, not `audio`.
15. Voice `Cherry` is for `qwen3-omni-flash-realtime`; qwen3.5-omni-flash uses `Tina`.
16. PipeWire holds audio hardware exclusively â€” route container ALSA through the PipeWire socket, not direct `hw:0,0`/`plughw`.
17. `webrtc-audio-processing` `bundled` build needs `meson >= 1.1`, `ninja-build`, `pkg-config`, `libclang-dev`, `python3-pip`. Debian bookworm's apt `meson` 1.0.1 breaks; `pip3 install meson --break-system-packages`. `libclang-dev` required by `bindgen`.
18. **livekit-wakeword 0.1 `predict()` needs ~2s of audio** â€” shorter returns 0.0 for every classifier. Predicting on a 100ms chunk silently always misses.
19. **livekit-wakeword classifier is position-sensitive** under training framing â€” the kassandra.onnx expects ~800ms of leading silence before the word inside the 2s window. Without it, isolated audio scores 0.008; with it, 0.66. Predict buffer must always be `[lead silence] + [speech]`.
